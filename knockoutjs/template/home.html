Ну что, давайте начнем

Наше приложение запускается в /index.html в документе.

Если вы откроете его, вы увидите следующий код:

<script data-main="scripts/app/app.js" src="scripts/vendor/require.js">

data-main эта директива запускает Require JS. Он будет отвечать за загрузку нашего javascript в приложение. В нашем случаи Require JS первым загружает файл /app.js, с которого и начинает работать наше приложение:

define(["viewModel"], function(viewModel) {

viewModel.launch();

});

viewModel.launch() запускает viewModel, который будет обрабатывать главную страницу. Если вы откроете index.html, Вы заметите, что есть только футер и хедер, но нет контента. Это потому-что Knockout JS берет всю ответственность за него на себя. Контент каждой страницы представлен следующим кодом:

<div data-bind="template: { name : 'home', templateUrl: 'template' }, visible : showTemplate('home')"></div>

<div data-bind="template: { name : 'routing', templateUrl: 'template' }, visible : showTemplate('routing')"></div>

<div data-bind="template: { name : 'controlling', templateUrl: 'template' }, visible : showTemplate('controlling')"></div>

<div data-bind="template: { name : 'services', templateUrl: 'template' }, visible : showTemplate('services')"></div>

<div data-bind="template: { name : 'docs', templateUrl: 'template' }, visible : showTemplate('docs')"></div>

template{} указывает, что весь контент будет отображаться в пределах этого тега.

это приложение не ограничивается только главной страницей, поэтому нам нужен способ для переключения наших views, в пределах одной страницы (не забывайте, мы создаем одностраничное приложение)
