Routes

You are still on the same page.

Yes, the view has changed, you see new text... But there wasn't any server call to get new page, no traffic was spent to load it. Javascript (in our case Angular JS) has produced new view itself.

Let's find out, how does it work

First of all you'll want to take a look at buttons, we are using for navigation (see /index.html):

<li active-link="active"><a href="#/home">Getting started></a></li>
<li active-link="active"><a href="#/routing">Routing</a></li>
<li active-link="active"><a href="#/controlling">UI Control</a></li>
<li active-link="active"><a href="#/services">Services</a></li>
Note the anchor URLs, used in links.

Then, if you open scripts/app.js, you'll see the routing provider, which manages all the routing, including history implementation:

$routeProvider.

when("/home", {templateUrl: "views/home.html", controller: app.HomeController }).

when("/routing", {templateUrl: "views/route.html", controller: app.RouteController}).

when("/services", {templateUrl: "views/service.html", controller: app.ServiceController}).

when("/controlling", {templateUrl: "views/controlling.html", controller: app.UIController}).

when("/docs", {templateUrl: "views/supplementary/docs.html", controller: app.DocsController}).

otherwise({redirectTo: "/home"});

The mechanism is simple - Angular JS uses pattern matching to find out which template should be put to app body.

You will also note controllers, which are added to an appropriate template (e.g. home.html has app.HomeController)
